**Plain-language purpose:** Read a real processor as a sequence of constrained design decisions instead of a collection of feature names.

## Terms introduced here

| Term | Meaning |
|---|---|
| open-source core | processor design whose implementation is publicly inspectable |
| design tradeoff | improvement in one objective paid for by another resource or risk |
| bottleneck | resource that currently limits useful progress |
| sensitivity | how strongly an output changes when one design input changes |
